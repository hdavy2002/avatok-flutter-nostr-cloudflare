## AVA-CORE-6 — `scripts/check_ava_reason.sh`

Bash, macOS-compatible (plain POSIX grep, fixed-string `-F` matching so marker
punctuation is literal — no GNU-only flags). Behavior:

- Resolves repo root from the script's own location, then scans `worker/src` and
  `consumers/src` for the direct model-call markers: `openrouter.ai`, `env.AI.run(`,
  `.AI.run(`, `api.openai.com`, `generativelanguage.googleapis.com`, `api.deepseek.com`.
- Any file path containing `ava_reason` is always tolerated (the helper modules).
- Paths listed in `scripts/ava_reason_allowlist.txt` (one repo-relative path per line,
  `#` comments and blanks ignored) are tolerated.
- Output: offenders printed as `file:line:content` to stderr with a remediation note;
  **exit 1** if any non-allowlisted offender exists, else a one-line OK summary and
  **exit 0**. Made executable (`chmod +x`, committed as mode 100755).
- NOT wired into CI (builds are manual-only by owner decision); for agents/humans and
  future CI.

**Verification performed:**
- With the seeded allowlist → `OK … exit=0`.
- With the allowlist emptied → `FAIL — 63 direct model-call site(s) … exit=1`, offenders
  listed correctly. (63 = individual match lines across the 31 seeded files.)
