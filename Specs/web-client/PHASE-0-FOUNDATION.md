## Steps (do them in order)

### 1. Scaffold
- `mkdir web && cd web`, `npm create astro@latest` (empty template, TypeScript strict), then add integrations: `npx astro add cloudflare react tailwind` (accept config writes).
- Set `astro.config.mjs` output to `'server'` or `'hybrid'` with the Cloudflare adapter so static pages prerender and islands run on the edge. Prefer **`hybrid`**: static by default, opt-in SSR per page.
- Pin Node 20 in `package.json` `engines`.

### 2. Token exporter (the design-parity backbone)
- Write `scripts/export-zine-tokens.mjs`: read `../app/lib/core/ui/zine.dart`, regex-parse every `static const <name> = Color(0xAARRGGBB)` and the numeric `static const double` tokens (`r, rSm, rField, rBadge, bw, bwLg`, durations), and the shadow offsets.
- Emit **`src/styles/tokens.css`** (`:root { --zine-paper:#F9F7ED; ... }`) and a **`tailwind.zine.cjs`** fragment exporting `theme.extend` (`colors`, `borderRadius`, `borderWidth`, `boxShadow`, `fontFamily`, `transitionDuration`).
- Convert Flutter `0xAARRGGBB` → CSS: if `AA=FF` emit `#RRGGBB`, else emit `rgba()`.
- **Shadows:** emit hard offsets with **no blur**: `shadow: '6px 7px 0 var(--zine-ink)'`, `shadowSm: '3px 3px 0 var(--zine-ink)'`, `shadowXs: '2px 2px 0 var(--zine-ink)'`, `shadowFocus: '5px 6px 0 var(--zine-blueInk)'`, `shadowError: '5px 6px 0 var(--zine-coral)'`.
- Wire it: add `"prebuild": "node scripts/export-zine-tokens.mjs"` and run it in `predev` too. Import `tailwind.zine.cjs` into `tailwind.config.ts`.
- **Verify:** the emitted `paper` is `#F9F7ED`, `lime` `#BFEB56`, `coral` `#FE674C`, `ink` `#231B14`, `blueInk` `#007D7F`. If not, fix the parser before continuing.

### 3. Fonts
- Self-host Fredoka, Nunito, Space Mono as `woff2` under `src/fonts/`. Add `@font-face` in `global.css` with `font-display: swap`. Map in Tailwind `fontFamily`: `display: Fredoka`, `body: Nunito`, `mono: 'Space Mono'`. Set `<body>` default to `body` font, headings to `display`.

### 4. apiClient
- `lib/config.ts`: `export const API_BASE = import.meta.env.PUBLIC_API_BASE ?? 'https://api.avatok.ai'`.
- `lib/apiClient.ts`: a typed `request<T>(path, {method, body, auth})` that prefixes `API_BASE`, JSON-encodes, attaches `Authorization: Bearer <jwt>` when `auth` is passed, throws a typed `ApiError {status, error}` on non-2xx. Add named helpers used by A–E: `getExplore`, `getListing`, `getCreator`, `getLiveNow`. Add a `ws(path, token)` helper that builds the `wss://` URL for the WebSocket endpoints. **Do not call any endpoint not in MASTER-PROMPT §4.**

### 5. Clerk provider + the shared GuestGate (every other phase depends on this)
- `lib/clerk.tsx`: a `<ClerkIsland>` wrapper exposing `useAuthToken()` (returns the session JWT, or null for an anonymous visitor) and a `SignInButton`. Read the Clerk publishable key from `PUBLIC_CLERK_PUBLISHABLE_KEY`. Keep it lazy — only hydrate where an island needs auth.
- **Build the reusable `GuestGate`** here (it lives in the foundation because B, C, D, and E all use it — see MASTER-PROMPT §4b). API: `requireGuestAuth(): Promise<string>` (resolves to a JWT, opening the modal only if no session) and a `<GuestGate onAuthed={...}>` modal component. The flow it runs:
  1. If a Clerk session exists → resolve its JWT immediately, no UI.
  2. Else open a `zine`-styled modal: collect **email** → `POST /api/id/email/start` → user enters OTP → `POST /api/id/email/verify` → `POST /api/identity/guest` → resolve the new JWT.
  3. Errors (bad code, resend) handled inline.
- Document `requireGuestAuth` / `GuestGate` props in `components/README.md` — this is a contract C/D/E/B code against. Do **not** put any per-feature logic in it; it only produces an authenticated session.

### 6. Shared component kit (mirror `zine_widgets.dart`)
- Read `app/lib/core/ui/zine_widgets.dart`. Build React equivalents in `components/`, styled only with Tailwind/token classes: **Button** (lime primary, coral destructive, ink border, hard shadow, press = `shadowPressed`), **Card**, **ListingTile** (poster card used in grids), **Avatar** (uses the Cloudflare image-transform URL pattern `/cdn-cgi/image/format=avif,quality=60,width=N,fit=cover/<path>`), **Pill/Sticker**, **Sheet/Modal**, **Spinner**. Export a typed API and **document each component's props in `components/README.md`** — that file is the contract A–E rely on.

### 7. Base layout + Nav shell
- `layouts/Base.astro`: `<head>` with charset, viewport, font preloads, a slot for per-page OpenGraph/Twitter meta, `global.css` + `tokens.css`. Body = `paper` background, nav, `<slot/>`, footer.
- `components/Nav.astro`: a static top bar (logo, and an empty links area marked `<!-- NAV LINKS: populated in Phase Z -->`). Do **not** hardcode feature links — Phase Z adds them once routes exist.

### 8. Prove the pipeline
- `npm run build` must pass. Run `npm run preview` (or `wrangler pages dev`) and confirm a blank `paper`-colored page in the right fonts.
- Add a `README.md` documenting: how to run, the token exporter, the kit, the apiClient, and the **rule that A–E own only their own dirs**.
