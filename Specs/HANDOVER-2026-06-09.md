## 5. Diagnostics system (PostHog) — how to pull logs by email

- **Client**: `AvaLog` (now in shared package `ava_diagnostics/`) buffers lines + forwards
  each to a `sink`. `Analytics.init()` wires the sink to PostHog `capture('diag_log',
  {tag, level, line, log_app, session})`. `identify(npub)` runs on every app open
  (`chat_list.dart`) so logs attach to the person. **No PII in PostHog** (keyed by npub).
- **Shared package**: `ava_diagnostics` (pure Dart). Any future app: add the path dep,
  `AvaLog.I.app = '<appkey>'`, wire the PostHog sink, then `AvaLog.I.log('tag','msg')`.
- **PostHog**: EU `https://eu.i.posthog.com`, project **139917**, key in `app/lib/core/analytics.dart`.

### To diagnose (given two emails)
1. Resolve each email → npub:
   ```sh
   # sha256 hex of the trimmed, lowercased email:
   node -e "import('crypto').then(c=>console.log(c.createHash('sha256').update('EMAIL'.trim().toLowerCase()).digest('hex')))"
   npx wrangler@4 d1 execute avatok-meta --remote --command \
     "SELECT npub FROM profiles WHERE email_hash='<hash>' ORDER BY updated_at DESC LIMIT 1"
   ```
2. Query PostHog (via the PostHog MCP / HogQL) by `distinct_id IN (npub1, npub2)`:
   ```sql
   SELECT timestamp, distinct_id, properties.tag, properties.level, properties.line, properties.session
   FROM events WHERE event='diag_log' AND distinct_id IN ('<npub1>','<npub2>')
   ORDER BY timestamp
   ```
   Interleave both users by time to see both sides of a DM/call.
